Были изучены базовые приёмы работы с github репозиторием (хранилищем кода) из коммандной строки:

  • git add . - добавление в коммит всех файлов из текущей папки и вложенных папок рекурсивно.
  • git commit -m "комментарий" - сохранение коммит из добавленых ранее в него файлов с комментарием
  • git pull --rebase origin - скачивание и применение к локальному репозиторию всех изменений из удалённого (внешнего) репозитория
  • git push origin - отправка всех сохранённых коммитов в удалённых репозиторий

  • Нажимаем Ctrl+F

Узнала о свойстве, которое используется на сайтах со 100% шириной. В этом случае, сайт хорошо смотрится как на больших, так и на маленьких дисплеях. Нарисуем макет, для большей наглядности (пример 1).

  1. header - занимает всю ширину экрана;
  2. sidebarL - занимает 200px, фиксированная ширина;
  3. sidebarR - также 200px, фиксированная ширина;
  4. content - занимает блок, неограниченный по ширине;
  5. footer - во всю ширину экрана браузера.

Как видим, на большом мониторе, header и footer растягиваются на всю ширину браузера, sidebarL и sidebarR остаются на месте (пример 2).

Так выглядит 15" и 24" монитор. Строчки растягиваются, читать на большом экране не так удобно (пример 3).

Создадим html и css файлы (пример 4,5).

Итак, что же делать?
Сделаем ограничение ширины сайта воспользуясь правилом. Уменьшение размера браузера будет происходить до определенного момента, затем появится полоса прокрутки. С ней можно просматривать сайт.

Для дива wrapper или "обертка" пропишем минимальную ширину. Блок будет занимать пространство не менее 650px (Пример 10).

Когда ширина дойдет 650px появится полоса прокрутки и можно будет смотреть содержимое сайта (Пример 11)
А для того, чтобы на большим мониторах, содержимое не растеклось по всему экрану, зададим другое правило. То есть, ограничим максимальную широту сайта. Для дива wrapper или "обертка" пропишем еще и максимальную ширину (Пример 12).

На очень большом экране монитора ширина сайта не растягивается больше 1100px, остается на месте. Все как и должно быть, все отлично читается и смотрится (Пример 13). Мы достигли своей цели, сайт можно будет смотреть на разных мониторах.
